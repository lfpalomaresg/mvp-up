# Ejemplos de uso MVP-UP

> ⚠️ Skill personal: referencia rutas de `~/claude_workspace/AI_OS/` — no transferible tal cual.

## Ejemplo 1 — Express sobre app con repo (piloto: blindbeds-appcc)

**Usuario:** `/mvp-up appcc`

**Flujo esperado:**
1. Intake: repo `github.com/lfpalomaresg/blindbeds-appcc` (ruta local del clon), sin URL pública aún, ficha `AI_OS/PROJECTS/` + memoria. Etapa: MVP funcionando end-to-end. Pregunta clave → objetivo de valor (p.ej. "primer cliente de pago").
2. Express para "MVP funcionando" → técnica · producto/UX · comercial · marketing/hype. Con agrupación: 2-3 agentes en paralelo (técnica+seguridad, producto/UX, comercial+marketing).
3. Consolidación → matriz → roadmap H1/H2/H3.
4. Primera pasada: no hay comparación; el informe queda como línea base en `AI_OS/PROJECTS/_escalado/appcc/`.
5. TOP-5 presentado → el operador autoriza → se ejecuta o encola.

## Ejemplo 2 — Full sobre producto facturando

**Usuario:** "quiero la auditoría completa de Blindbeds Supply antes de enseñársela a un segundo hotel"

**Flujo esperado:**
1. Intake detecta "auditoría completa" → confirmar modo full («son 10 agentes, pasada cara, ¿OK?»).
2. 10 agentes en paralelo, cada uno con su bloque de `dimensiones.md`.
3. Ponderación por objetivo "vendible a más hoteles" → pesan más operativa, técnica, legal y comercial.
4. Informe + comparación si existe pasada anterior + TOP-5.

## Ejemplo 3 — Servicio sin software (el spa de un hotel boutique)

**Usuario:** "escala el servicio del spa con mvp-up"

**Flujo esperado:**
1. Intake: no hay repo → técnica y seguridad-código = N/A; legal y operativa usan variante servicio.
2. Express: comercial · operativa · económica · mercado (3 agentes agrupados).
3. El agente económico trabaja con los datos de la propuesta de precios; lo que falte = N/D + lista de datos a pedir al hotel.

## Contraejemplo — cuándo NO usar MVP-UP

- "Revisa la seguridad de blindbeds.com" → eso es `tool-site-audit` directamente.
- "¿Está listo para deploy?" → `tool-quality-gate`.
- "Audita mis skills y mi sistema" → `tool-mantenimiento-sistema`.
MVP-UP es para la vista 360° de UN producto con roadmap de escalado, no para chequeos puntuales.
